## მონაცემთა დამუშავება (Cleaning)
###  1.გამოტოვენბული მნიშვნელობების(NA) მქონე სვეტების მოშორება
ისეთი სვეტები სადაც ძალინ ბერვი NA არის სრულიად უნდა ამოვშალოთ, რადგან მათი რაიმე სხვა მნიშვნელობით ჩანაცვლება ზედმეტ noise-ს შემოიტანს, რამაც რეალურად შეიძლება უფრო გააფუჭოს მოდელი ვიდე გაგვიუმჯობესოს. გავოვიყენე threshold > 80, რის შედეგადაც ფოტოზე ნაჩვენები პირველი ოთხი სვეტი დაიდროპება რადგან მეოთხე სვეტის პროცენტულობა არის Fence  -> 80.051370.

### 2. NA შევსენა
  სანამ უშუალოდ შევსებაზე გადავალთ საჭიროა კატეგორიული ცვადებისა და რიცხვით ცვადების გამოყოფა, რომ სხვანაირად შევავსოთ.
  თავიდან რიცხვითი ცვლადები შევავსე საშუალო არითმეტიკულით, მაგრამ მერე მივხვდი, რომ უმჯობესია მედიანით შევსება, რადგან ის უფრო მდგრადია outliers-ების მიმართ. 
  კატეგორიულ ცვლადებს ვავსებს მოდით(Mode), ყველაზე ხშრად შეხვედრილი მონაცემებით შევსება უფრო ლოგიკურია ვიდრე 'None'-ებით, რადგან უცნობი კატეგორია მოდელს უფრო დააბნევს.

### 3. დუბლიკატებისა და broken დატას შემოქმება
  თუ სადმე გვაქვს დუბლიკატები ან აშკარად არასწორი მნიშვნელობები, კერძოდ, უარყოფით მნიშვნელობები, რომელიც წესით არ უნდა იყოს უარყოფითი, შეუძლებელი მნიშვენლობები ა.შ. ეგენი შეგვიძლია დავდროპოთ, მაგრამ დუპლიკატებზე შემოწმებისას დამიბრუნა np.int64(0) -> ესეიგი არ გვაქ დუბლიკატები. უცნაური და არასწორი მიშვნელობებიც არ შემხვედრია, ამიტომ შეგვიძლია ჩავთვალოთ, რომ ამ ეტაპზე დადროპვას მოვრჩით.

## როგორ ვაკეთებდით ზღვრების დასტაბილურებას
საჭიროა სწორი  threshold-ის პოვნა, რათა შევინარჩუნოთ სწორი ინფორმაცია და ხმაური სწორად მოვიშოროთ.
ერთმანეთთან კორელაციის დროს threshold გაზრდა ძალაინ სახიფათა რადგან არის შანსი, მოდელში დარჩება ძალიან ბევრი მსგავსი ცვლადი. შედეგად, მოდელი გახდება ზედმეტად რთული და მოხდება Overfitting (ტრენინგზე კარგი შედეგი, ტესტზე - ცუდი).
ხოლო ძალიან შემცირებთ  ჩვენ წავშლით ბევრ ისეთ ცვლადს, რომლებსაც რეალურად სასარგებლო ინფორმაცია მოჰქონდათ. შედეგად მივიღებთ Underfitting-ს (მოდელი იქნება ზედმეტად მარტივი და ვერ ისწავლის კანონზომიერებებს). ამიტომ threshold-ებს და RFE სვეტების რაოდენობას ვარეგულირებდი. Training-ის შედეგებზე დაყრდნობით — val_rmsle-ს მიხედვით.
